# NOVA AI - All-in-One AI Web Platform

A lightweight, modular, and production-ready AI platform designed for low-resource systems. It features Chat AI, Image Generation, and Video Editing capabilities.

## Features

- **Authentication**: User registration, login, and secure email OTP password reset.
- **Chat AI**: Powered by local Ollama (TinyLlama/Mistral).
- **Image Generation**: Optimized Stable Diffusion (CPU).
- **Video Editing**: Text-to-command FFmpeg processing.
- **Frontend**: Clean, dark-themed UI built with HTML/CSS/VanillaJS.
- **Backend**: High-performance FastAPI with SQLite database.

## Prerequisites

- Python 3.10+
- FFmpeg (added to system PATH)
- Ollama (installed and running)

## Setup Instructions

1.  **Clone the repository:**
    ```bash
    git clone <repo_url>
    cd nova_ai
    ```

2.  **Create a virtual environment:**
    ```bash
    python -m venv venv
    source venv/bin/activate  # Linux/Mac
    venv\Scripts\activate     # Windows
    ```

3.  **Install dependencies:**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Configure Environment Variables:**
    - Copy `.env.example` to `.env`:
      ```bash
      cp .env.example .env
      ```
    - Edit `.env` and add your details (especially Email for OTP).

5.  **Setup AI Models:**
    - **Ollama**:
      - Install Ollama from [ollama.com](https://ollama.com).
      - Run `ollama serve`.
      - Pull the model: `ollama pull tinyllama`.
    - **Stable Diffusion**:
      - The system will attempt to download the model on first run. Ensure you have internet access.

6.  **Run the Application:**
    - Navigate to the `nova_ai` directory:
      ```bash
      cd nova_ai
      ```
    - Run the server:
      ```bash
      uvicorn backend.main:app --reload
      ```

7.  **Access the Platform:**
    - Open `http://127.0.0.1:8000` in your browser.

## API Documentation

The API documentation is automatically generated by FastAPI.

## Deployment

To deploy on a local server (localhost):
1.  Ensure all prerequisites are met.
2.  Set `DEBUG=False` in `.env`.
3.  Run with Uvicorn:
    ```bash
    uvicorn backend.main:app --host 0.0.0.0 --port 8000
    ```
